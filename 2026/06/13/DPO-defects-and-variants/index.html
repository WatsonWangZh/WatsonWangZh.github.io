本文整理 DPO（Direct Preference Optimization）的基本推导、常见缺陷及其若干变体。DPO 将 RLHF 中的 reward model 与强化学习优化过程合并为一个监督式的二分类目标，因此在 LLM 对齐中被广泛使用。但 DPO 的简洁性也带来一些隐含假设：所有偏好对被视为同等强度，优化目标只关心相对概率差，且 reward 由 policy 与 reference model 的 log-ratio 间接定义。DPOP、SimPO、LD-DPO、IPO、ORPO、KTO 等方法大多可以看作对这些假设的修正。

DPO Defects

DPO 的优点是形式简单、训练稳定、不需要显式训练 reward model，也不需要在线 RL。但在实际训练中，DPO 至少有以下几个问题。

Reference Dependence

DPO 的 reward 由 log-ratio 定义：

因此训练时需要同时计算 $\pi_{\theta}$ 与 $\pi_{\text{ref}}$ 的 log probability。工程上，这会增加显存和计算开销；理论上，reference model 的质量也会影响 reward 的尺度与方向。

Unbounded Margin

DPO 使用 logistic loss。当 $h_{\theta}$ 变大时，loss 会持续鼓励 margin 增大。若偏好数据存在噪声，或者某些 pair 太容易区分，模型可能把 log-ratio 推得过大，导致过优化或偏离原模型。

Positive Probability Decrease

DPO 只关心相对差值 $h_{\theta}$。因此只要 rejected response 的概率下降得更多，即使 preferred response 的概率也下降，DPO loss 仍可能变小。换言之，DPO 不直接保证 $\pi_{\theta}(y_w|x)$ 上升。

Binary Preference Strength

标准 DPO 把所有 preference pair 当作同等强度的二元偏好。但实际标注中，有的 pair 差距很小，有的 pair 差距极大。如果不区分偏好强度，训练信号会被压缩成单一的 chosen/rejected 标签。

Length Bias

Data Format Limitation

DPO 需要 pairwise preference，即同一个 prompt 下至少有一个 chosen 和一个 rejected。实际系统中更常见的数据可能是单条回答的 thumbs-up / thumbs-down、用户是否采纳、是否触发安全拦截等 binary feedback。DPO 无法直接利用这些单样本标签。

DPOP

DPOP 通常指 DPO-Positive。该方法针对 DPO 的一个 failure mode：DPO 可能通过降低 rejected response 的概率来扩大 margin，同时 preferred response 的概率也被降低。

动机

实践选择

DPOP 适合用于 chosen 与 rejected 差别较小的数据集，例如回答只有局部修改、推理步骤只有少量差异、摘要或代码补全中编辑距离较小的 preference pair。若数据 pair 差别很大，DPOP 的收益可能不如 SimPO 或 ORPO 明显，但它仍然是修正 DPO 正样本下降问题的直接方案。

SimPO

LD-DPO

LD-DPO（Length Desensitization in Direct Preference Optimization）直接针对 DPO 的长度敏感问题。它认为 DPO 的隐式 reward 与 response length 之间存在强相关，容易将 verbosity 当作人类偏好。

动机

IPO

IPO（Identity Preference Optimization）将 DPO 中无界增长的 preference margin 改为固定目标 margin，从而缓解 DPO 的过优化问题。

IPO 适合偏好数据较小、噪声较多、担心 DPO 过拟合的情况。它的训练目标更像 regression，稳定性较好，但 target margin 的设定会影响学习强度。若数据非常干净且 pair 数量充足，标准 DPO 或 SimPO 可能已经足够。

KTO

KTO（Kahneman-Tversky Optimization）从 prospect theory 出发，将人类对 gain/loss 的非对称感知引入 alignment loss。与 DPO 最大的不同是：KTO 不要求 pairwise preference，只需要单条样本的 desirable / undesirable 标签。

动机

真实业务数据中，pairwise preference 昂贵且难收集。相比之下，单条回答的好坏反馈更容易获得，例如用户点赞、点踩、是否点击、是否采纳、是否被安全系统判定为违规。KTO 试图直接利用这种 binary feedback。

实践选择

KTO 适合缺少成对偏好数据、但有大量单样本反馈的场景。若已经有高质量 pairwise preference，DPO/SimPO/IPO 仍然直接；若线上积累的是 thumbs-up/down 或安全审核标签，KTO 的数据利用方式更自然。

Comparison

从 DPO 的缺陷出发，可以将上述方法理解为不同方向的修正：

  1. DPOP 解决的是 DPO 不保证 preferred response 概率上升的问题；
  2. SimPO 解决的是 reference model 开销与长度归一化问题；
  3. LD-DPO 解决的是 DPO 对 response length 过度敏感的问题；
  4. IPO 解决的是 DPO margin 无界、容易过优化的问题；
  5. ORPO 解决的是 SFT 与 preference optimization 两阶段训练复杂的问题；
  6. KTO 解决的是 DPO 必须依赖 pairwise preference data 的问题。

如果偏好数据干净、显存充足，DPO 仍然是很好的 baseline。如果希望降低工程成本，可优先考虑 SimPO 或 ORPO。如果训练后回答明显变长，应重点检查 LD-DPO 或 SimPO 的长度归一化思路。如果担心过拟合或数据噪声，IPO 的固定 margin 更稳。如果只有单样本好坏标签，KTO 比强行构造 preference pair 更自然。
